## 调用流程

### 1. 用户查询家电状态

```
用户: "空调开了吗？"
  │
  ▼
OfficeClaw 智能体
  │
  ├── LLM 分析意图 → 需要查询家电状态
  │
  ├── 匹配工具 → GreenSkillTool (get_appliance_status)
  │
  ├── 调用 fetch_appliance_status()
  │     │
  │     ▼
  │   GET http://101.34.223.37/api/status
  │     │
  │     ▼
  │   FastAPI 读取 appliance-status.json
  │     │
  │     ▼
  │   返回 JSON 数据
  │
  ├── format_status_text() 格式化
  │
  ├── LLM 生成自然语言回复
  │
  ▼
用户: "空调正在运行，制冷模式，设定26°C，室温27.5°C。"
```

### 2. 用户控制家电开关

```
用户: "把电视关掉"
  │
  ▼
OfficeClaw 智能体
  │
  ├── LLM 分析意图 → 需要关闭电视
  │
  ├── 匹配工具 → ToggleDeviceTool (toggle_appliance)
  │
  ├── 解析参数 → device="电视", action="off"
  │
  ├── 调用 toggle_device("电视", "off")
  │     │
  │     ├── _resolve_device("电视") → "tv"
  │     │
  │     ├── PATCH /api/status/tv/status     → {"value": "off"}
  │     │
  │     └── PATCH /api/status/tv/status_text → {"value": "已关闭"}
  │           │
  │           ▼
  │         FastAPI 更新 appliance-status.json
  │           │
  │           ▼
  │         返回 {"message": "电视已关闭"}
  │
  ├── LLM 生成回复
  │
  ▼
用户: "电视已关闭。"
```

## API 接口速查

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/status` | 获取全部设备状态 |
| `GET` | `/api/status/{device}` | 获取指定设备状态 |
| `PUT` | `/api/status/{device}` | 替换指定设备全部状态 |
| `PATCH` | `/api/status/{device}/{field}` | 修改指定设备单个字段 |

**device 可选值：** `air_conditioner`、`refrigerator`、`lights`、`tv`

## 部署说明

```bash
# 1. 安装依赖
cd api && pip install -r requirements.txt

# 2. 启动 API 服务
APPLIANCE_JSON_PATH=/usr/share/nginx/html/appliance-status.json \
  uvicorn appliance_api:app --host 0.0.0.0 --port 8090

# 3. Nginx 配置 (已包含 /api/ 反向代理)
sudo nginx -s reload

# 4. 安装 green-skill 到 OfficeClaw
# 上传 green-skill.zip 到 OfficeClaw 平台
```
